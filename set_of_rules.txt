
## 🛠️ Metadata Curation Rules for `label_disambiguator.py`-Processed Entries

### 1. **Use Only Established Categories**
- Stick to your defined metadata fields:
  - `#genres`, `#book_format`, `#provenance`, `#writing_style`, `#narrative_structure`, `#emotional_tone`, `#character_traits`, `#book_setting`, `#reading_mood`, `#reading_level`, `#publication_period`, `#themes`, `#length`, `#pacing`, `#perspective`, `#subject`, `#movement`
- **Do not invent new categories** (e.g., no `#artistic_style`, `#character_arc`, etc.)

---

### 2. **Preserve Existing Labels Exactly as They Appear**
- If a label has a suffix (e.g. `thoughtful-et`, `legacy-t`, `heroism-s`), **do not alter or remove it**.
- These suffixes were added by `label_disambiguator.py` to resolve cross-category collisions and are polished later in CalibreSynapseTUI.
- **Never duplicate or reformat existing suffixed labels.**

---

### 3. **Add New Labels Only Where Needed**
- You may enrich a category **only if it feels underrepresented**.
- When adding new labels:
  - **Do not add suffixes** (even if other labels in the same category have them).
  - Example: If `themes` already includes `family-t`, you may add `mentorship` — not `mentorship-t`.

---

### 4. **Respect Category–Label Alignment**
- Ensure each label belongs to the correct category:
  - `superhero` → `#genres`
  - `comics` → `#book_format`
  - `usa-p` → `#provenance`
  - `all ages` → `#reading_level`
  - `2010s` → `#publication_period`
  - `third-person` → `#perspective`
  - etc.

---

### 5. **Avoid Overwriting or Replacing Existing Labels**
- If a base label (e.g. `themes`) was overwritten by suffixed labels (e.g. `family-t`, `identity-t`), **restore the base label** by re-adding it:
  ```bash
  --field "#themes:family-t, identity-t, legacy-t, mentorship"
  ```
  → Here, `mentorship` is added without a suffix, and the base label `themes` is preserved.

---

### 6. **Comments Field Should Be Expressive but Not Redundant**
- Use the `comments` field to summarize emotional tone, narrative arc, and thematic focus.
- Avoid repeating labels verbatim — instead, reflect their meaning in prose.


### ✅ Refined Metadata Guidelines

#### 🧩 Field vs Label
- **Fields** are categories like `series`, `publisher`, `perspective`.
- **Labels** are values inside fields: e.g. `comedy`, `1960s`, `adult`, `action`, `cheerful`, `clever`.

### Book Format
- Never use "book" as a book format because each and every book has their own format, it could be novel, non-fiction,  biography, comics, manga, graphic novel, art book, history book, etc.

#### 🔢 Label Quantity & Format
- Use **up to 10 labels per field**, only if necessary.
- **One-word labels preferred**, but **multi-word labels allowed** when needed (e.g. `alien worlds`, `childhood imagination`).
- Use **commas**, not "and": e.g. `introspective, emotional`.

#### 📚 Format-Specific Rules
- If the file is **manga or comics**, append **(manga)** or **(comics)** to the `series` name.
- If it's a **book**, use the plain series name.
- Use:
  - `#mangagenre:` for manga
  - `#genres:` for comics and other formats
- use proper manga categorizaion: e.g. shojo, shonen, seinen, mecha, josei, slice of life etc.


#### 🕰️ Publication Period
- Use either:
  - Decades: `1980s`, `2000s`
  - Descriptors: `Ancient`, `Classics`, `Modern Classics`, `Modern`
- You may combine: e.g. `"publication_period": ["2000s", "modern"]`

#### 🌍 Provenance
- Use **country of origin**, not publisher.
- Use lowercase for `usa`, `uk`; full names for others: `Japan`, `France`, `Canada`.
- For multi-national works: `Canada, France, Norway`

#### 📏 Length
Use qualitative descriptors:
- `short` — under ~250 pages
- `medium` — ~250–400 pages
- `medium to long` — ~400–600 pages
- `long` — ~600–800 pages
- `very long` — ~800–1000 pages
- `massively long` — 1000+ pages or structurally epic

#### 🧠 Subject
- Focus on **universal concepts or motifs**.
- You can also focus on things like: how the homicide is conducted in case it's a crime story, so for example by using labels like (acid, bullying, clocktower murder, blackmail, betrayal, cosplay affair, coming of age, creative process, crime investigation, cult murders etc.) and it extends to other descriptor depending on the book genre and content.
- Avoid:
  - Specific characters or factions
  - Plot-specific events
  - In-universe tech or locations unless conceptually broad
field should reflect universal concepts, motifs, or domains that the work engages with — not specific characters, plot points, or in-universe elements. It’s about the intellectual or thematic territory the book occupies, not its narrative cast


#### 🧒 Reading Level
Use: `children`, `teen`, `teen and up`, `young adult`, `adult`, `all ages`, `mature`

#### 👁️ Perspective
- Use only: `first-person`, `third-person`

#### 🧭 Movement
- Use cultural or literary movement if applicable.
- Leave blank if none fits.

#### 🗺️ Book Setting
- Use **generalized descriptors**: `estate`, `village`, `urban district`, `countryside`
- Exception: include **recognized landmarks or regions** (e.g. `Tower of London`, `Paris`)
- You may prefix with country: `Japan, forest`

#### ✍️ Description Guidelines
- Always include:
  - Format intro: manga or comics
  - Colorful HTML emoticons
  - Info about the book or collection
  - Biographical details about the author
